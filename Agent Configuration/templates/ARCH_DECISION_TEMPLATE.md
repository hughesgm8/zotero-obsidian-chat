
# Purpose
This file contains a template to be used when updating ARCHITECTURAL_DECISIONS.md with a log of major architectural or UX decisions. This is to help both you and the user understand the state of the project, the logic behind decisions made, and what other options might be worth considering.

# Template

```markdown
## Decision: [Simple description]

### Why This Matters
[User benefit in plain language]

### Options We Considered
1. [Option A]: Good for [benefit], bad for [drawback]
2. [Option B]: Good for [benefit], bad for [drawback]

### Why We Chose This
- [Reason 1 that matters to end users]
- [Reason 2 that affects maintainability]
- [Reason 3 that impacts future changes]

### What Could Change
If [condition changes], we might switch to [alternative]
```
