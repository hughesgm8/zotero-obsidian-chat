## [~Session] Fix text selection and copy-with-sources in chat view

### Change 1
- **File Modified**: `styles.css`
- **Change**: Added `user-select: text` and `cursor: text` to `.zotero-chat-message-content`
- **Logic Update**:
  ```
  .zotero-chat-message-content {
    + user-select: text
    + cursor: text
  }
  ```
- **Reason**: Obsidian applies `user-select: none` broadly to sidebar UI elements. Without overriding it, users couldn't highlight or copy text from their own message bubbles.

### Change 2
- **File Modified**: `src/chat-view.ts`
- **Change**: Copy button on assistant messages now includes the sources list in the clipboard output
- **Logic Update**:
  ```
  on copy click:
    text = msg.content
    if msg.sources:
      text += "\n\n**Sources:**\n"
      for each source: text += "- title — authors (year)\n"
    clipboard.write(text)
  ```
- **Reason**: `msg.content` is only the LLM prose body — sources are stored separately in `msg.sources`. Previously clicking copy omitted them entirely, which was frustrating when pasting into notes.
